# Issue 011 — End-to-end pipeline CLI

**Labels:** `pipeline`, `sprint-02`
**Spec refs:** `spec/MVP.md` §F-11; `spec/TECHNICAL_PLAN.md`
**Estimate:** 0.5 day
**Depends on:** #010

## Goal

Wire all pipeline stages into the `python -m horror_story run` command with full flag
support. After this issue, the full *Pigeons from Hell* pipeline can be run end-to-end.

## Acceptance criteria

- [ ] `python -m horror_story run --story <path> --out <dir>` runs all stages
- [ ] `--scene <id>` re-runs only the named scene and re-renders from that point
- [ ] `--dry-run` prints the execution plan without writing any files
- [ ] `--validate` validates all artifacts in the output dir against their schemas
- [ ] `--seed <n>` overrides the manifest seed
- [ ] `--width` / `--height` override render resolution (for dev mode)
- [ ] Per-scene errors are caught and summarized; exit code 1 if any failed
- [ ] Stage progress logged as `[STAGE] scene_id → output_path` to stdout
- [ ] End-to-end test: run the mini-story fixture (3 scenes) and assert final MP4 exists
- [ ] `pytest tests/test_cli.py` passes (requires FFmpeg; skip if absent)
- [ ] `mypy --strict` passes

## Tasks

1. Implement full `cli.py` orchestration using the stage functions from previous issues.
2. Implement `--dry-run` by collecting the plan without calling stages.
3. Implement `--validate` using `schemas.validate_all(output_dir)`.
4. Add end-to-end test with 3-scene mini-story fixture.
5. Write `tests/fixtures/mini-story.txt` (hand-written 3-scene horror snippet).

## Notes

- The mini-story fixture should be ≤ 300 words and contain at least one dialogue line.
- All tests that invoke FFmpeg must use `pytest.mark.skipif(not ffmpeg_available(), ...)`.
