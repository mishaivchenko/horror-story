# Issue 012 — GitHub Actions CI

**Labels:** `ci`, `sprint-01`
**Spec refs:** `CLAUDE.md` §CI; `spec/constitution.md` §Definition of done
**Estimate:** 0.25 day
**Depends on:** #001

## Goal

Validate and wire GitHub Actions to run the test suite on every push and PR. The workflow
files already exist in `.github/workflows/`; this issue's job is to confirm they work
once the scaffold (issue #001) is in place and to fix anything that doesn't.

## Acceptance criteria

- [ ] `ci.yml` runs green on a passing scaffold commit (0 or more tests, no errors)
- [ ] CI installs Python 3.11, installs the package in dev mode (`pip install -e ".[dev]"`)
- [ ] CI runs `pytest --cov=horror_story --cov-fail-under=80`
- [ ] CI runs `mypy --strict src/`
- [ ] CI runs `python -m horror_story validate-schemas`
- [ ] `spec-lint.yml` runs `markdownlint` on spec Markdown files
- [ ] All workflows are valid YAML (run `yamllint .github/workflows/*.yml` locally)
- [ ] FFmpeg-dependent tests are skipped gracefully in CI via `pytest.mark.skipif`

## Tasks

1. After #001 lands: push to `main` and verify both workflow runs succeed.
2. Fix any YAML errors or missing dependencies discovered in the first run.
3. Confirm `validate-schemas` sub-command stub exists in the CLI (added in #001).
4. Document any required secrets or environment variables (there should be none for MVP).

## Notes

- Workflow files (`.github/workflows/ci.yml`, `spec-lint.yml`) and issue templates already
  exist on disk. Do not rewrite them unless a functional bug is found.
- Use `ubuntu-latest` runner with `actions/cache` for pip.
- FFmpeg is not installed on the runner; all FFmpeg tests must use `pytest.mark.skipif`.
