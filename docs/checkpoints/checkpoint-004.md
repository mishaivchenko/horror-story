## Known Technical Debt

1. **`adapters/base.py` is partially dead.** `TTSAdapter` was moved to `adapters/tts/base.py` and re-exported via `base.py`. The remaining four ABCs (`ImageAdapter`, `MotionAdapter`, `AudioAdapter`, `TypographyAdapter`) live in `base.py` with 0% test coverage. They are forward-declarations, not implementations — acceptable at this stage but need to migrate to their own `adapters/<type>/base.py` modules before the corresponding issues.

2. **`output/fixtures/` is absent.** CI workflow (Issue #012) expects `output/fixtures/` to contain schema-validated artifact examples. Currently no fixtures exist at that path. The CI `validate` step will be a no-op until fixtures are added.

3. **`cli.py` `_cmd_run` and `_cmd_validate` are stubs.** Tested via `pytest` argument parsing only; functional wiring blocked until #011.

4. **`AGENTS.md` and `CODEX.md` are untracked.** Intentional workflow documentation sitting at repo root, not `.gitignore`'d, not committed. Should either be tracked or explicitly ignored before #005.

5. **`models.py` line 48 (mood tie-break) is uncovered.** Minor, non-blocking. The `max()` lambda tie-break path requires two moods with equal keyword counts — not covered by any test.

## Top 5 Risks Before Issue #005 (Image Adapter)

1. **`Pillow` is a dev dependency, not a runtime dependency.** If `MockImageAdapter` or `ImageAdapter` uses PIL for image generation/validation, it must remain in `[dev]` until a real provider is integrated. Risk: accidental promotion to `dependencies` bloats the runtime footprint.

2. **No `output/fixtures/` for CI schema validation.** Issue #012 CI job validates artifacts in `output/fixtures/`. Without at least one `keyframe.json` fixture post-#005, that CI step will silently pass vacuously. Risk: schema contract goes untested in CI until someone notices.

3. **`adapters/base.py` migration timing.** `ImageAdapter` ABC lives in `base.py` with no subpackage. Issue #005 should move it to `adapters/image/base.py` for consistency with the TTS pattern. Delaying this creates a precedent of uneven module structure across adapters.

4. **`AGENTS.md` / `CODEX.md` unresolved.** These files are untracked. If a Codex review runs from a fresh clone it may miss workflow context. Low probability, moderate confusion risk.

5. **Mock `text_secondary` field is reversed word order.** This placeholder will silently pass schema validation indefinitely. Risk: if real translation is not gated by a spec change and acceptance test, the mock value leaks into visual stages (typography, subtitles) producing garbage output that still validates.
